|This Site|

This site was coded completely from scratch. When I initially started, I knew that I wanted to be able to edit it from any device, have it sync automatically with cloud storage, and not have to pay any money to do either. For my storage method, I originally chose Google Drive (as I am a heavy Google user). Next, I needed a method to host websites on Google Drive. Turns out that a man named Amit Agarwal wrote a niftly little Google Script (tutorial here) that did just that. The script has you upload a zip file containing the webpage files, and then creates a public folder in your Drive containing those files. It then generates a web link using the folder name, and gives that link to you. I had already used the free subdomain redirection service co.nr, so I linked chromeoverload.co.nr to the Google Drive website link.

However, a few years later I noticed that the Google Drive hosting trick had become defunct, and the links no longer worked. Thus, I decided to move everything to Github and edit it all online from Github itself. Thus, the site in its current iteration was created.

An early screenshot of the homepage with its original "vintage red" CSS style.

|How I Edit This Site|

To edit this site, I had originally wanted to use a cross-platform IDE that I could load from any device. As a Google user, I looked toward Chrome-based IDEs, specifically ones on the Chrome Web Store. I chose the IDE Puresolid IDE because it had the best autcomplete, text highlighting, and overall interface. I can install this Chrome App on any computer. But, as a backup, I wanted a browser-based editor that I could use from even my mobile devices. I found Editey, a Drive App/browser based editor that runs fairly smoothly on my iPad. And with that, I was set!

However, since Github has its own built-in online IDE/coding interface, I just everything there now.
